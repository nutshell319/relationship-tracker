# 感情发展追踪网站 — 设计规格书

> 日期：2026-07-15 | 状态：已确认

## 1. 项目概述

### 1.1 目标

构建一个纯前端网站，通过分析微信聊天记录（WeChatMsg 导出 JSON），自动追踪和可视化与心仪对象的感情发展过程。

### 1.2 核心约束

| 约束 | 说明 |
|------|------|
| 数据隐私 | 聊天原始数据仅存浏览器内存，不上传任何服务器 |
| 公网访问 | 网站代码托管 GitHub Pages，任何人可访问 URL |
| 零后端 | 所有分析在浏览器端完成，无服务端逻辑 |
| 低成本 | 仅 AI API 按量计费（几分钱~几毛钱/次分析） |

---

## 2. 技术方案

### 2.1 技术栈

| 层 | 选型 | 原因 |
|----|------|------|
| 框架 | Vue 3 CDN | 比原生 JS 省 DOM 操作，无需构建工具 |
| 图表 | ECharts CDN | 关系型图表丰富，热力图/日历图开箱即用 |
| AI API | DeepSeek（V4 Pro / Flash） | 用户现有 API Key，OpenAI 兼容格式 |
| 存储 | localStorage | 仅存分析结果（不含原始聊天内容） |
| 部署 | GitHub Pages | 纯静态托管，免费 |

### 2.2 数据流

```
用户拖入 JSON → FileReader（浏览器内存）→ 数据验证器
  → 统计引擎计算
  → AI 分析器（若已配 Key）
  → 渲染可视化
  → 分析结果存入 localStorage
```

- 原始聊天 JSON 仅存内存，关闭页面即消失
- AI API 调用时仅发送抽样/摘要，不发送完整记录
- localStorage 只存统计结果 + AI 总结，不含原始对话
- GitHub 仓库只有代码，无任何聊天数据

## 3. 页面结构

单页应用，3 个视图通过 JS 切换：

### 3.1 数据导入页（入口）

- 拖拽/选择 WeChatMsg 导出的 JSON 文件
- 解析验证，显示聊天概况（消息总数、时间跨度、双方昵称）
- API Key 配置入口（DeepSeek，存 localStorage）
- 确认后进入分析

### 3.2 分析仪表盘（核心）

2×2 卡片网格 + AI 总结区：

| 位置 | 图表 | 类型 |
|------|------|------|
| 左上 | 感情趋势曲线 | ECharts 折线图，X=时间 Y=亲密度评分 |
| 右上 | 消息量统计 | ECharts 柱状图，按日/周聚合 |
| 左下 | 高频词云 | 词云组件 |
| 右下 | 互动热度热力图 | ECharts 热力图，星期几 × 小时段 |
| 全宽 | AI 总结卡片 | 自然语言阶段描述 |
| 扩展 | 互动天平 | 对称柱状图，双方消息量/主动发起对比 |
| 扩展 | 聊天日历热力图 | GitHub 贡献图风格，每日聊天密度 |
| 扩展 | 称呼演变追踪 | 时间轴标注称呼变化节点 |
| 扩展 | Emoji 使用趋势 | 折线图，常用 emoji 频率变化 |
| 扩展 | 深夜聊天分析 | 22:00-02:00 占比趋势 |
| 扩展 | 年度聊天报告 | 长图翻页式总结 |

### 3.3 关系时间线

- 纵向时间轴
- AI 识别的阶段标注（初识期/暧昧期/…）
- 里程碑事件卡片（日期 + 标题 + 摘要 + 重要性评分）
- 感情趋势背景色带

## 4. AI 分析引擎

### 4.1 三轮递进策略

**第 1 轮 · 月度切片扫描**
- 按月切分聊天记录 → 每月首/中/尾抽样 → AI 打分（亲密度/话题深度/暧昧指数）→ 月度摘要一句话
- Token：约 2K/月 × N 个月

**第 2 轮 · 全局汇总判定**
- 所有月度摘要 + 打分 → AI → 输出：关系阶段划分 + 各阶段特征 + 整体趋势
- Token：约 3K~5K

**第 3 轮 · 重要事件挖掘**
- 阶段划分 + 各阶段原始抽样 → AI → 识别关键事件 + 时间 + 摘要 + 重要性评分
- Token：约 3K~5K

### 4.2 输出 JSON Schema

```json
{
  "phases": [
    {
      "name": "初识期",
      "start": "2025-03",
      "end": "2025-05",
      "intimacy": 35,
      "description": "..."
    }
  ],
  "trend": { "direction": "up", "slope": 0.8 },
  "milestones": [
    {
      "date": "2025-04-15",
      "title": "第一次深入对话",
      "importance": 8,
      "summary": "..."
    }
  ]
}
```

### 4.3 降级策略

- 未配 API Key 或调用失败 → 自动降级为纯统计模式
- 页面顶部提示 + 提供重试按钮
- 统计图表正常展示，仅缺少自然语言总结和事件标注

## 6. 异常处理

| 场景 | 处理方式 |
|------|----------|
| 文件格式不匹配 | 提示 + 展示期望格式示例 |
| AI API 调用失败 | 自动降级纯统计 + 重试按钮 |
| 聊天记录只有一人 | 提示"未检测到双方互动" |
| 文件过大（>50MB） | 显示大小 + 估算时间 + 确认对话框 + 进度条 |
| localStorage 满 | 提示清理 + 手动清除按钮 |
| API Key 无效 | 提示检查 Key + 降级纯统计 |

### 隐私保障

- 一键清除按钮：清空所有 localStorage 数据
- 关闭页面即清除内存中的原始聊天数据
- GitHub 仓库中 `.gitignore` 排除任何可能含数据的文件
